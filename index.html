Orders

Create orders and link them to payments. Order creation is an important step as it helps you associate every payment with an order. Orders and payments can be created in one-go or seperately.

Orders

Create an Order

POST /orders
OrderInput
Content-Type: string
in header

application/json

expand: string[]
in query

Expand the object creation

Scenario Definition How-to
POST /orders?expand[]=payment Create objects Order and Payment in one go. This is recommended for easing the no. of API calls. Include Payment object in the request body
POST /orders Only Order will be created. This gives the flexibility to link to a Payment object at a later time Exclude Payment object in the request body. After an Order is created, link Order to Payment by POST /orders/{id}/payment
Array values passed as multiple parameters: ?expand=aaa&expand=bbb
