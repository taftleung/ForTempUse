Inbound API

Payment QR Code Creation API

POST /payment/qrCode

This API creates QR Code token or image with the aim of making payment. Once this API request is submitted from merchants, HSBC payment platform will return QR Code token/image to Merchant regarding to the PayNOW / SGQR specification. Merchant then displays the QR Code to Buyer, who scans and enters further information to complete the payment.

Outbound API

Payment Status Notification API

POST /<Client_Defined_Endpoint>

Payment status will be returned to Merchant by asynchronous callback once HSBC receives a payment request. After HSBC payment platform completes reconciliation with HKFPS and receives payment result, HSBC will push the result back to Merchant by calling this API.

Implementation HSBC will trigger this API call and defines the interface with OpenAPI standard. Merchant is required to provide implementation.

Retry Mechanism If no success response is received, up to 4 retries will be triggered in every 2 minutes. Maximum 5 calls including the 1st attempt.

Endpoint Definition Field notifyUrl from Payment Request API will be used as URL endpoint of the corresponding transaction.

qrCodeRequestModel: object

merId: string (up to 18 chars)

Merchant ID

  • Distributed by HSBC to merchant for identifying each merchant's identity
txnRef: string (up to 25 chars)

Unique ID referred to a specific transaction

  • Required Merchant to generate a unique ID for each transaction in alphanumeric format with up to a maximum of 25 characters
  • This txnRef will be embedded in the QR code generated, and will be shown to Payer for confirmation during scan and pay of QR code. If PayNow scheme is selected by the payer to settle the payment, HSBC will return this txnRef field in outbound API call to merchant for payment status notification
txnChannel: string 01, 02 (up to 2 chars)

Transaction Channel

Possible Value Definition
01 POS
02 e-Commerce / m-Commerce
txnTime: string (up to 20 chars)

Time of sending out this request transaction

  • Client system time. Formatï¼š yyyy-MM-dd'T'HH:mm:ssZ
  • Time zone is determined by field country
qrExpiry: string (up to 10 chars)

QR code Expiry

  • Formatï¼š yyyymmdd
  • When payer scan the QR code and the transaction date is later than QR expiry date, payer bank will not proceed the PayNow payment to merchant
  • Optional field. If this field is not provided, it means no expiry date will be specified in the generated QR code. Otherwise, the value must be equal or later than todayâ€™s date
payMethod: string SGQR, PAYNOW (up to 10 chars)

Payment Method

Possible Value Definition
SGQR Singapore QR Code for E-Payments
PAYNOW PayNow QR Code Payment
amtEditInd: string Y, N (up to 1 chars)

Amount Editable Indicator

  • To indicate if the amount can be edited by payer or not Please refer to PayNOW spec for details. Can specify Y/N here
  • Notice: This option is only for PayNow but not applicable to SGQR
  • Conditional field. Required when payMethod = "PAYNOW"
qrOption: string 01, 02 (up to 2 chars)

QR Code Option for PayNow. Data type returned in field qrCode from response message.

  • Notice: This option is only for PayNow but not applicable to SGQR
  • Conditional field. Required when payMethod = "PAYNOW"
Possible Value Definition
01 Return Raw QR Data
02 Return Base64 encoded QR image in PNG format
country: string SG (up to 2 chars)

Country Code

Possible Value Definition
SG Singapore
  • Format: ISO 3166-1 alpha-2
currency: string SGD (up to 3 chars)

Payment Currency

Possible Value Definition
SGD Singapore Dollar
  • Format: ISO 4217 Alpha
amount: integer 1 â‰¤ x â‰¤ 999999999999

Payment Amount

  • Format: Eliminate punctuation and sign, support 2 decimal places according to ISO 4217, e.g. $10.50 = 1050
notifyUrl: string (up to 128 chars)

URL provided by Merchant for returning status used by Payment Status Notification API

goodsDes: string (up to 128 chars)

Description of goods. Optional field.

posMachineId: string (up to 36 chars)

Unique ID of a POS device.

  • Conditional field. Required when txnChannel = "01"
employeeId: string (up to 36 chars)

ID of a staff member who handles a specific POS transaction.

  • Conditional field. Required when txnChannel = "01"
